## 摘要
大语言模型在生成响应之前，通过在数十层网络中传播激活值来处理提示词。我们探讨指令提示词中包含的任务相关信息是否可以压缩为单个激活向量，并重新注入模型以替换原始的 token 序列？我们证明，通过使用在目标大语言模型中间层提取并注入到早期层的学习型激活加权和，这一目标是可以实现的。压缩后的向量保留了任务相关信息，相对于完整提示词处理，其准确率下降不到 2%。除了实际应用意义（包括在不重新处理原始 token 序列的情况下减少固定指令提示词的单次查询计算量）外，我们的分析还揭示了大语言模型激活空间的结构：(i) 中间层表示可以有意义地迁移到早期层，这表明信息编码方式具有一定程度的跨层兼容性；(ii) 单个激活向量编码了可量化且可恢复的语义信息量；(iii) 激活值的加权和是一种鲁棒的表示压缩器。

## Abstract
Large language models process prompts by propagating activations through dozens of layers before generating a response. We ask whether the task-relevant information contained in an instruction prompt can be compressed into a single activation vector and re-injected into the model, replacing the original token sequence? We show this is achievable using a learned weighted sum of activations extracted at an intermediate layer and injected at an early layer of the target LLM. The compressed vector preserves task-relevant information, incurring an accuracy drop of under $2\%$ relative to full prompt processing. Beyond its practical implications, including reducing per-query computation for fixed instruction prompts without reprocessing the original token sequence, our analysis reveals structure in the activation space of LLMs: (i) mid-layer representations transfer meaningfully to early layers, suggesting a degree of cross-layer compatibility in how information is encoded; (ii) a single activation vector encodes a quantifiable and recoverable amount of semantic information; (iii) a weighted sum of activations is a robust representation compressor.

### 1. 核心问题与整体含义（研究动机和背景）
在 LLM 推理中，系统提示词、指令或 Few-shot 示例往往是重复且固定的，但每次查询都需要重新计算这些 Token 的激活值，造成计算资源浪费。虽然 KV-Cache 能缓解部分问题，但无法缩短序列长度。
本研究的核心问题是：**能否将整个指令提示词的任务相关信息压缩成一个单一的激活向量（Patch Vector），并将其注入模型，从而完全替代原始 Token 序列？** 这不仅是一个工程优化问题，更是在探究 LLM 内部激活空间如何编码语义信息。

### 2. 论文提出的方法论
研究者提出了一个“提取-压缩-注入”的三步框架：
*   **提取（Extraction）：** 从预训练 LLM 的中间层 $m$ 提取提示词 $p$ 的隐藏状态序列 $H^{(m)}(p)$。
*   **压缩（Compression）：** 提出两种压缩函数将序列映射为单个向量 $v$：
    1.  **权重 MLP (W-MLP)：** 核心思想是“加权和”。使用一个小型 MLP 为每个 Token 的激活值预测一个标量权重 $w_i$，最终向量 $v = \sum w_i h_i$。这种方法基于线性表示假设。
    2.  **Transformer 压缩器 (TC)：** 使用一个小型 Transformer 编码器对全序列进行注意力处理，输出一个类 [CLS] 的压缩向量。
*   **注入（Injection）：** 在模型的早期层 $e$（通常 $e < m$），用压缩向量 $v$ 替换一个占位符 Token（如 `¿`）的激活值。模型随后基于此向量和用户查询生成回答。

### 3. 实验设计
*   **模型：** Llama 3.1-8B-Instruct（主实验）、Ministral-8B、Qwen 3.5-4B、Llama 3.2-1B。

    2.  **ARC-Easy：** 标准科学常识多选题基准，用于测试更复杂的语义压缩。
*   **基准（Benchmark）：**
    *   **Full Prompt：** 完整输入原始提示词（性能上限）。
    *   **Masked Prompt：** 仅保留占位符但不注入压缩向量（性能下限）。
*   **对比方法：** 对比了 W-MLP 和 TC 两种压缩方案。

### 4. 资源与算力
*   **硬件：** 论文明确提到实验运行在 **NVIDIA RTX A5500** GPU 上。

### 5. 实验数量与充分性
*   **实验规模：** 涵盖了 4 种不同架构和规模的模型，进行了跨任务（ID/OOD）的验证。

    *   **占位符与位置：** 测试了不同占位符 Token 及注入位置对性能的影响。
    *   **信息容量：** 测试了单个向量同时编码多个任务的能力。
*   **评价：** 实验设计客观、公平，通过对比 Full/Masked 基准清晰地展示了压缩效果。

### 6. 主要结论与发现
*   **高效压缩：** W-MLP 压缩方案在任务指令上仅导致 **<2%** 的准确率下降。
*   **泛化性：** 简单的 W-MLP 比复杂的 TC 泛化性更好，TC 容易在特定任务上过拟合。
*   **跨层兼容性：** 实验证明 LLM 的中间层表示可以有意义地迁移回早期层，暗示了模型内部编码的某种线性结构。
*   **语义聚焦：** 通过可视化权重发现，W-MLP 会自动为具有核心语义的 Token（如关键词、标点符号）分配更高权重。
*   **高容量：** 单个激活向量在 Llama 3.2-1B 等小模型中也能同时编码多达 9 个任务而无明显性能损失。

### 7. 优点
*   **轻量级：** 无需微调原始 LLM，仅需训练一个极小的 MLP。
*   **可解释性：** W-MLP 的权重直接揭示了哪些 Token 对任务最重要，具有天然的特征归因属性。
*   **推理效率：** 显著减少了处理长系统提示词时的 Token 数量和注意力计算开销。

### 8. 不足与局限
*   **有损压缩：** 压缩本质上是有损的，对于极高信息密度或需要复杂推理的提示词，性能可能会进一步下降。
*   **白盒限制：** 该方法需要访问模型的内部激活值，无法应用于闭源 API（如 GPT-4）。
*   **任务范围：** 目前主要在短提示词和知识检索类任务上验证，尚未在长文档摘要或复杂逻辑推理任务中充分测试。
*   **预计算开销：** 压缩过程本身需要一次部分前向传播，因此该方法更适用于会被多次重复调用的固定提示词场景。
