## 摘要
大语言模型（LLMs）在嵌入生成领域已得到广泛探索。虽然近期研究表明，上下文学习（ICL）通过在输入前添加少量任务相关示例，能有效增强 LLM 的表示能力，但由于序列长度增加，这会导致显著的 token 开销。在这项工作中，我们提出了 EPIC，一种新型的基于嵌入的上下文提示训练策略，旨在利用 ICL 生成高质量嵌入，同时减轻训练和推理阶段的计算负担。该方法将离散的文本示例替换为对应的连续嵌入，这不仅促使 LLM 在对比学习过程中对齐语义相关的文本对，还要求模型将示例嵌入作为上下文提示的一部分进行解析。因此，经 EPIC 训练的模型在推理阶段，无论是否使用上下文提示，均能表现出卓越的嵌入性能。全面实验表明，我们的方法在 MTEB 基准测试中取得了新的最先进（SOTA）结果，超越了仅在公开检索数据上训练的前沿模型。广泛的消融研究进一步验证了该机制的有效性和必要性。

## Abstract
Large language models (LLMs) have been widely explored for embedding generation. While recent studies show that in-context learning (ICL) effectively enhances the representational capability of LLMs by prepending a few task-related demonstrations, it causes substantial token overhead due to the increased sequence length. In this work, we propose EPIC, a novel embedding-based in-context prompt training strategy that leverages ICL to generate high-quality embeddings while reducing computational burden during both training and inference. This approach replaces discrete text demonstrations with their corresponding continuous embeddings, which not only encourages the LLM to align semantically-related text pairs during contrastive learning, but also requires the model to interpret demonstration embeddings as part of the in-context prompt. Consequently, EPIC-trained models achieve excellent embedding performance both with or without in-context prompts at inference time. Comprehensive experiments demonstrate that our method establishes new state-of-the-art results on the MTEB benchmark, surpassing frontier models trained solely on publicly available retrieval data. Extensive ablation studies further validate the effectiveness and necessity of our mechanism.

### 1. 核心问题与整体含义（研究动机和背景）
论文的核心问题在于解决**大语言模型（LLM）作为文本编码器时，上下文学习（ICL）带来的高昂计算开销**。
*   **背景**：近期研究发现，在输入 Query 前添加少量任务相关的示例（Demonstrations）能显著提升 LLM 生成嵌入向量的质量。
*   **动机**：传统的 ICL 依赖于离散的文本示例，这会大幅增加输入序列的长度，导致训练和推理时的 Token 开销巨大，且增加了推理延迟，在检索增强生成（RAG）等对延迟敏感的场景中难以实用。
*   **目标**：提出一种既能保留 ICL 增强效果，又能显著降低计算负担的训练策略。

### 2. 方法论：EPIC 核心思想与技术细节
论文提出了 **EPIC (Embedding-based In-Context Prompt Training)**，其核心思想是**用连续的嵌入向量（Continuous Embeddings）替代离散的文本示例**。

*   **核心流程**：
    1.  **示例压缩**：将任务相关的“查询-段落对” $(Q_i, P_i)$ 分别通过 LLM 编码，提取其 [EOS] 标记的隐藏状态。
    2.  **特征映射**：通过一个轻量级的 MLP 层（两层线性变换加 GELU 激活）对这些嵌入进行处理，将其映射到与模型输入兼容的潜在空间。
    3.  **提示构建**：将处理后的连续向量 $q_i, p_i$ 与任务指令 $I$ 拼接，形成嵌入式上下文提示。
    4.  **输入拼接**：将这些连续向量直接插入到当前待编码 Query 的 Token Embedding 序列中。
*   **训练策略**：
    *   **监督对比学习**：使用 InfoNCE 损失函数。在训练过程中，从同一个 Batch 内随机采样正样本对作为 ICL 示例。
    *   **随机采样**：训练时随机选择 0 到 5 个示例，使模型同时具备“有提示”和“无提示”下的高性能。
*   **推理优化**：在推理阶段，特定任务的示例嵌入可以**预先计算并存储**，后续只需拼接向量即可，无需重复计算示例部分的 Attention。

### 3. 实验设计
*   **数据集**：使用 E5 数据集的公开部分进行训练（约 150 万个样本），涵盖 ELI5、HotpotQA、MS-MARCO 等多种检索任务。
*   **Benchmark**：**MTEB (Massive Text Embedding Benchmark)**，包含 56 个数据集，涵盖检索、重排序、聚类、分类、STS 等 7 类任务。
*   **对比方法**：
    *   **传统模型**：BGE-large-en-v1.5, UAE-large-v1。
    *   **LLM 编码器**：E5-Mistral, GritLM, LLM2Vec, Anchor, MGH, bge-en-icl 等。
*   **基础模型**：Qwen2.5-7B, Mistral-7B, LLaMA-3.1-8B。

### 4. 资源与算力
*   **硬件**：实验使用了 **NVIDIA A100 80GB GPU**。
*   **训练时长**：
    *   Mistral-7B 训练 1000 步。
    *   Qwen2.5-7B 和 LLaMA-3.1-8B 训练 800 步。
    *   使用 LoRA 微调（Rank=64），梯度累积设为 8，模拟 Batch Size 为 512。
*   **效率对比**：EPIC 的训练开销仅比 Baseline 增加约 19%，而传统文本 ICL 增加超过 60%。

### 5. 实验数量与充分性
*   **实验规模**：在全量 MTEB（56 个数据集）上进行了主实验，并在 26 个数据集的子集上进行了详尽的消融实验。
*   **消融实验覆盖面**：
    *   不同模型规模（1B, 3B, 8B）的扩展性分析。
    *   连续向量数量的影响（证明 1 个向量代表一个文本已足够）。
    *   Prompt 格式与压缩策略的对比。
    *   与 Soft-prompt（可学习 Token）的对比。
    *   不同注意力机制（因果 vs 双向）和池化策略的对比。
*   **充分性评价**：实验设计非常全面，不仅验证了性能，还深入分析了推理延迟、Token 长度和“注意力汇（Attention Sink）”现象，实验结果具有较高的客观性和说服力。

### 6. 主要结论与发现
*   **性能领先**：EPIC 在仅使用公开数据训练的模型中达到了 **SOTA**，在 Mistral-7B 基础上 MTEB 平均分达到 66.37。
*   **双重增强**：即使在推理时不提供任何 ICL 示例，EPIC 训练的模型也优于普通对比学习训练的 Baseline。
*   **高效性**：相比文本 ICL，EPIC 在推理时可降低高达 **70% 的延迟**，并显著减少 Token 消耗。
*   **缓解注意力汇**：分析发现 EPIC 训练能让模型减少对第一个 Token 的过度关注，使 [EOS] 标记能更有效地聚合全句语义。

### 7. 优点（亮点）
*   **创新性**：首次在嵌入模型中提出用连续向量替代离散文本作为 ICL 示例，打破了传统 Prompt 的形式限制。
*   **实用性强**：通过预计算示例嵌入，解决了 LLM 编码器在实际生产环境中延迟过高的痛点。
*   **鲁棒性**：模型在有无示例的情况下表现均很出色，具有极强的灵活性。

### 8. 不足与局限
*   **数据覆盖**：未对比使用大规模私有合成数据或 MTEB 相关数据训练的闭源模型（如 Gemini Embedding），这在一定程度上限制了绝对分数的上限。
*   **模型规模**：实验最高仅覆盖到 8B 参数规模，尚未在 30B 或 70B 等更大规模模型上验证其增益是否会持续扩大。
*   **理论解释**：虽然实验证明了有效性，但对于“为什么模型能理解自身生成的嵌入作为提示”的深层机理（Mechanistic Explanation）探讨尚浅。
