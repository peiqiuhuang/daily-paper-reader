## 摘要
大语言模型（LLM）智能体在实际任务中的表现，不仅取决于冻结的模型本身，还取决于其模型周围的框架（harness）：包括提示词、注入的知识、运行时控制和配置。在部署中，框架通常是唯一可用的杠杆，因此自动改进框架是在不触动权重的情况下提高性能的自然方式。难点不在于生成变更，而在于确定哪些变更真正起到了作用。自我生成的反馈存在噪声，表面的增益可能是测量偏差，或者是仅对微调任务过拟合的修改。我们提出了一种自演化的智能体框架，它将提议变更与归因（crediting）分离开来：由语言模型诊断失败并提议补丁（patches），而所有的采样、测量和显著性测试则由确定性代码负责，因此从结构上保证了每一次被认可的改进都是可信的。补丁填充了一个门控的、分类的质量多样性存档（GSME），该存档以修改所针对的（位置 x 原因）病理特征（pathology）为键，而非其修复的任务，这是一种抗过拟合的归纳偏置；泛化性在演化结束后才评分的封闭测试集上进行衡量。在七个领域中使用冻结的开源权重模型，框架在训练集中被选择，并在封闭测试中进行一次评分；其认可的增益为 +9 到 +15.5 个百分点，并保留了 86-147% 的训练增益，证明了其具有泛化性而非过拟合。胜出的补丁追踪的是模型的主要病理特征，而非其规模或家族：更换模型可能会改变病理特征和补丁，而相同的“病理-补丁”匹配在两个模型家族中重复出现。可迁移的是“诊断-归因”循环，而非任何特定的框架。

## Abstract
An LLM agent's real-task performance is shaped as much by the harness around its model as by the frozen model itself: its prompts, injected knowledge, runtime control, and configuration. In deployment the harness is often the only lever available, so improving it automatically is the natural way to raise performance without touching the weights. The hard part is not generating changes but knowing which one truly helped. Self-generated feedback is noisy, and an apparent gain can be a measurement artifact or an edit that merely overfits the tasks it was tuned on. We present a self-evolving agent-harness framework that separates proposing changes from crediting them: a language model diagnoses failures and proposes patches, while all sampling, measurement, and significance testing are owned by deterministic code, so every credited improvement is trustworthy by construction. Patches populate a gated, categorical quality-diversity archive (GSME) keyed on the (WHERE x WHY) pathology an edit addresses rather than the tasks it fixes, an anti-overfitting inductive bias; generalization is measured on a sealed test scored only after evolution. Across seven domains with a frozen open-weight model, the harness is train-selected and scored once on a sealed test; its credited gains there are +9 to +15.5pp and retain 86-147% of the training gain, evidence they generalize rather than overfit. The winning patch tracks the model's dominant pathology, not its size or family: changing the model can change the pathology and the patch, while the same pathology-to-patch match recurs across two model families. What transfers is the diagnose-and-credit loop, not any specific harness.

#### 1. 核心问题与研究动机
论文关注的是如何自动优化大语言模型（LLM）智能体的“框架”（Harness）。框架是指模型权重之外的所有部分，包括系统提示词、工具定义、控制循环、知识注入和运行配置。
*   **研究背景**：在实际部署中，模型权重往往是冻结的（如闭源 API 或微调成本过高），优化框架成为提升性能的唯一手段。
*   **核心挑战**：现有的自动优化方法存在两个痛点：一是**反馈噪声大**，难以区分真正的性能提升与测量误差（如沙盒波动）；二是**容易过拟合**，优化后的框架往往只在训练任务上表现好，无法泛化到新任务。

#### 2. 方法论：GSME 框架
论文提出了一种将“提议变更”与“认可信用”分离的自演化循环，核心技术包括：
*   **语义质量多样性存档（GSME）**：
    *   使用 **(Where × Why)** 二元组作为索引。`Where` 指修改的层面（提示词、知识、运行时、配置）；`Why` 指诊断出的病理特征（如：思维失控、过早结束、方法锁定等）。
    *   这种基于“病理特征”而非“任务 ID”的分类方式提供了一种抗过拟合的归纳偏置。
*   **三重门控机制（Three Gates）**：
    *   **有效性门控**：排除基础设施故障（如超时、崩溃），确保测量结果真实。
    *   **激活门控**：只有当补丁在运行中实际被触发时，才计算其贡献。
    *   **显著性门控**：采用**配对 2σ 显著性测试**。只有通过统计学验证的改进才会被记录，从而过滤掉随机噪声。
*   **演化流程**：
    1.  **诊断**：由一个更强的模型（如 Claude Opus）分析失败轨迹，识别病理特征。
    2.  **设计**：生成针对性的代码补丁。
    3.  **预检与筛选**：在小规模锚点任务上快速测试，剔除明显无效的补丁。
    4.  **评估与归因**：在训练集上运行并经过三重门控校验，优胜者进入 GSME 存档。

#### 6. 主要结论与发现

*   **可迁移性**：特定的框架补丁可能无法跨模型通用，但“诊断-归因”的演化循环是高度可迁移的。
*   **框架本质**：演化出的框架本质上是针对特定模型失败分布的“纠错补丁”。

#### 7. 优点
*   **高可信度**：通过统计学门控和密封测试，解决了智能体自优化领域长期存在的“刷榜”和“过拟合”疑云。
*   **可解释性**：GSME 存档提供了清晰的病理特征图谱，让开发者了解模型为什么失败以及补丁如何起作用。
*   **端到端自动化**：实现了从失败诊断到代码补丁生成的全流程自动化。

#### 8. 不足与局限
*   **验证器依赖**：该方法高度依赖于每个任务必须有可靠的自动验证器（Verifier）。如果验证器本身有漏洞，演化过程可能会“走捷径”利用这些漏洞。
*   **代理目标偏差**：优化的是短期任务的成功率，可能无法捕捉到长期稳健性、可维护性等软性指标。
*   **成本问题**：尽管有加速算法，但反复的大规模基准测试依然消耗大量的推理 Token 和计算资源。
