## 摘要
大语言模型 (LLMs) 在挖掘电子健康记录 (EHR) 方面展现出巨大潜力，能够通过对纵向临床信息进行推理来捕捉上下文丰富的患者轨迹。然而，利用 LLMs 处理结构化 EHR（例如标准化的诊断和药物代码）面临两个关键挑战。首先，将带有时间戳的 EHR 序列转换为纯文本可能会模糊时间结构和代码标识，从而削弱捕捉代码共现和纵向规律的能力。其次，与在队列上训练、学习跨患者共享且任务对齐的表示空间的预测模型不同，LLMs 通常应用于病例隔离的推理场景，即独立处理每个患者，而未能利用群体层面的模式。为了应对这些挑战，我们提出了 RePrompT，这是一个时间感知型 LLM 框架，它通过提示微调集成结构化 EHR 编码器，且无需修改底层架构。具体而言，RePrompT 循环地整合来自先前就诊的隐藏状态以保留纵向信息，并通过源自队列训练、任务对齐的 EHR 编码器的可训练提示标记注入群体层面的信息。在 MIMIC-III 和 MIMIC-IV 上的实验表明，RePrompT 在多个临床预测任务中始终优于基于 EHR 和基于 LLM 的基准模型。

## Abstract
Large Language Models (LLMs) have shown strong promise for mining Electronic Health Records (EHRs) by reasoning over longitudinal clinical information to capture context-rich patient trajectories. However, leveraging LLMs for structured EHRs (e.g., standardized diagnosis and medication codes) presents two key challenges. First, translating time-stamped EHR sequences into plain text can obscure both temporal structure and code identities, weakening the ability to capture code co-occurrence and longitudinal regularities. Second, unlike cohort-trained predictive models that learn a shared, task-aligned representation space across patients, LLMs are often applied in a case-isolated inference setting where each patient is processed independently without leveraging population-level patterns. To address these challenges, we introduce RePrompT, a time-aware LLM framework that integrates structured EHR encoders through prompt tuning, without modifying underlying architectures. Specifically, RePrompT recurrently incorporates latent states from prior visits to preserve longitudinal information, and injects population-level information through trainable prompt tokens derived from a cohort-trained, task-aligned EHR encoder. Experiments on MIMIC-III and MIMIC-IV demonstrate that RePrompT consistently outperforms both EHR-based and LLM-based baselines across multiple clinical prediction tasks.

*   **整体含义**：提出 RePrompT 框架，旨在不修改 LLM 底层架构的前提下，通过提示微调（Prompt Tuning）集成结构化编码器，赋予 LLM 时间感知能力和群体知识。

### 2. 方法论
RePrompT 采用了一种“软硬结合”的提示策略，包含三个核心模块：
*   **临床记录综合（Clinical Record Synthesis）**：利用强大的通用模型（如 DeepSeek-V3）将冗长、噪声大的出院小结与结构化医疗代码压缩成简洁的文本摘要，作为“硬提示”。
*   **状态循环提示微调（State-Recurrent Prompt Tuning）**：
    *   为了捕捉纵向轨迹，模型逐次处理就诊记录。
    *   将上一时间步 LLM 的隐藏状态（Hidden State）经过线性变换后，作为当前步的“软提示”输入，实现跨就诊的信息传递。
*   **结构化编码提示微调（Struct-Encoded Prompt Tuning）**：
    *   集成一个预训练的结构化 EHR 编码器（如 RETAIN）。
    *   该编码器从群体数据中学习医疗代码的共现和本体关系，生成的患者嵌入向量被注入 LLM 作为另一组“软提示”，提供任务对齐的群体级背景。
*   **预测与优化**：使用冻结的 Llama 3.1 1B 作为预测器，在其顶层添加分类头，通过二元交叉熵损失进行端到端优化。

### 6. 主要结论与发现
*   **性能领先**：RePrompT 在所有任务和数据集上均显著优于传统的 EHR 模型和现有的 LLM 提示方法。
*   **时间感知的重要性**：消融实验显示，移除“状态循环”模块会导致性能大幅下降，证明了显式建模时间依赖性对 EHR 任务至关重要。
*   **群体知识的价值**：通过结构化编码器注入的“软提示”弥补了 LLM 在处理离散医疗代码和群体模式上的短板。
*   **小模型胜大模型**：经过微调的小型 LLM（1B 参数）在特定临床任务上的表现优于零样本的大型通用模型。

### 7. 优点
*   **架构优雅**：无需修改 LLM 的内部参数，通过软提示实现了异构数据（结构化代码+非结构化文本）的深度融合。
*   **时间建模创新**：引入循环机制处理 LLM 的隐藏状态，解决了 LLM 处理长纵向序列时的“平铺”问题。
*   **模块化强**：可以灵活更换底层的 EHR 编码器或预测 LLM。

### 8. 不足与局限
*   **数据依赖性**：模型性能高度依赖于 EHR 数据的质量和出院小结的完整性，跨医疗系统的泛化性有待验证。

*   **任务覆盖**：目前主要集中在分类任务，尚未探讨其在生成式临床任务（如自动撰写病历）中的表现。
