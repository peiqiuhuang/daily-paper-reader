# Introduction to Mathematical Programming with Equilibrium Constraints (MPECs) and Bilevel Optimization 

# Louis Shuo Wang ∗

Department of Mathematics, Northeastern University, Boston, 02115, MA, USA. 

Abstract 

Our aim is to explain mathematical programs with equilibrium constraints (MPECs), motivate them through applications, present the main equivalent for-mulations of equilibrium constraints, and summarize the basic existence theory for optimal solutions. The central message is that an MPEC is an optimization problem whose feasible set is partly defined by another optimization, variational inequality, complementarity system, or equilibrium model. 

# 1 Introduction and Big Picture 

An MPEC is an optimization problem in which part of the feasible set is defined implicitly through an equilibrium problem. The upper level chooses variables ( x, y ) to optimize a global objective, while lower-level feasibility of y is determined by a vari-ational inequality, complementarity problem, or optimization problem parameterized by x.This structure arises naturally in economics, engineering design, transporta-tion, mechanics, robotics, and game theory. It generalizes bilevel programming and complementarity-constrained optimization. It also introduces major analytical chal-lenges: nonconvex feasible regions, possible loss of closedness, nonsmoothness, and multi-valued lower-level response maps. 1

# 3.2 Interpretation of the model ingredients 

1. f (x, y ) is the upper-level objective. 2. Z collects joint constraints on ( x, y ). 3. F (x, y ) defines the lower-level equilibrium condition. 4. C(x) is the lower-level feasible set for y given x.5. S(x) is the lower-level solution map, also called the reaction map. 

# 3.3 Bilevel programming as a special case 

If there exists a continuously differentiable function θ(x, y ) such that 

F (x, y ) = ∇y θ(x, y ),

then the lower-level VI is the first-order optimality condition for min

If θ(x, ·) is convex and an appropriate constraint qualification holds, the VI solution set coincides with the optimizer set of the lower-level problem. 

Remark 3.1 (Optimistic versus pessimistic bilevel modeling) . If the lower-level prob-lem has multiple optimizers, the condition y ∈ S(x) does not by itself specify how the lower-level solution is selected. The formulation above is the standard optimistic or 3strong version when the upper level is allowed to choose the most favorable y ∈ S(x). A pessimistic or weak version instead evaluates the upper-level objective against the least favorable lower-level optimizer.

# 3.5 Why MPECs are difficult 

The main difficulties are structural rather than cosmetic. 1. Nonconvexity. Even if each lower-level problem is convex, the graph of the solution map S can be highly nonconvex, so F is typically nonconvex. 2. Possible failure of closedness. Although Z may be closed, Gr( S) may fail to be closed unless the lower-level solution map has suitable closed-graph properties. 3. Nonsmoothness. Even when S(x) is single-valued, it may fail to be differentiable in x.4. Multivaluedness. The lower-level solution map may be set-valued, which creates ambiguity in the upper-level model and motivates optimistic and pessimistic Stackelberg interpretations. 45. Combinatorial structure. Complementarity constraints encode disjunc-tions. For example, 0 ≤ a ⊥ b ≥ 0 means either a = 0 or b = 0 or both. This creates a latent finite-union structure similar to disjunctive programming.

# 4.3 Misclassification minimization 

A minimum-misclassification hyperplane problem can be reformulated as a bilevel linear program by introducing lower-level variables that encode misclassification indicators through optimization.

In robotic motion with frictional contact, the dynamic constraints together with Coulomb friction and complementarity conditions lead to a complementarity system. # 5.3 SBCQ and closedness of the feasible region 

A central issue is whether KKT multipliers can be chosen in a uniformly bounded way along feasible sequences. # 5.4 Piecewise structure of the feasible set 

The complementarity conditions partition the feasible region into finitely many pieces according to which constraints are active and which multipliers are positive. Therefore, under suitable assumptions, the feasible set is a finite union of closed sets. This reveals the latent combinatorial structure of MPEC. For each complementarity pair 0 ≤ λi ⊥ ui ≥ 0, where ui = −gi(x, y ), the feasible alternatives are 

λi = 0 , u i ≥ 0 or λi ≥ 0, u i = 0 .

With ℓ complementarity pairs, this creates up to 2 ℓ active-set regimes. 

# 5.5 Affine variational inequalities and MPAECs 

If 

F (x, y ) = q + N x + M y, g(x, y ) = b + Ax + By, 

then the lower-level VI is affine and the resulting problem is often called an MPEC with affine equilibrium constraints, or MPAEC. In this case, each active-set piece of the feasible set is polyhedral, so the global feasible set is a finite union of polyhedra. This special case is much more structured and often serves as a first serious testbed for theory and algorithms. 

# 5.6 Merit-function reformulations 

Complementarity conditions can be converted into equations using an NCP function. A standard example is the Fischer–Burmeister function: 

ϕ(a, b ) = pa2 + b2 − (a + b).

For the KKT inequalities gi(x, y ) ≤ 0, define the nonnegative slack 

ui(x, y ) := −gi(x, y ) ≥ 0.

Then complementarity is equivalently written as 

ϕ(λi, u i(x, y )) = 0 , i = 1 , . . . , ℓ. 

This yields an equation-based reformulation of MPEC and motivates smooth or semismooth numerical methods. 

# 5.7 Implicit-program formulation 

If, for each x, the lower-level VI has a unique solution y(x), then the MPEC reduces to min  

> x

f (x, y (x)) s.t. x ∈ X, (x, y (x)) ∈ Z. 

A common sufficient condition for uniqueness is strict or strong monotonicity of F (x, ·)on C(x). 

Definition 5.1 (Monotonicity) . A mapping G : K → Rm is: 1. monotone if ( u − v)T (G(u) − G(v)) ≥ 0 for all u, v ∈ K,2. strictly monotone if the inequality is strict for u̸ = v,3. strongly monotone if there exists c > 0 such that (u − v)T (G(u) − G(v)) ≥ c∥u − v∥2 ∀u, v ∈ K. 

Proposition 5.2 (Existence and uniqueness for a VI) . Let K ⊂ Rm be nonempty, closed, and convex, and let G : K → Rm be continuous. If K is compact, then 

VI( G, K ) has a solution. If, in addition, G is strictly monotone, the solution is unique. If K is unbounded, existence requires an additional coercivity or growth condition; strong monotonicity is a common condition that yields both uniqueness and, under the standard VI hypotheses, existence. 

For complementarity problems on orthants, a related uniqueness condition is the uniform P condition. 

# 5.8 Constraint qualifications that imply SBCQ 

SBCQ is implied by more familiar nonlinear programming conditions, including: 1. Mangasarian–Fromovitz constraint qualification (MFCQ), 2. constant-rank constraint qualification (CRCQ), 3. linear independence constraint qualification (LICQ). 10 LICQ is strongest among these and yields uniqueness of the multiplier vector.

# 6 Existence of Optimal Solutions 

# 6.1 General principle 

Existence of a minimizer usually follows from two ingredients: 1. the feasible set is closed, 2. at least one lower level set of the objective is nonempty and bounded. If these hold and the objective is continuous, a minimizer exists by the Weierstrass theorem. Closed feasible set

# 6.2 Coercivity and bounded level sets 

A function ψ on a set S is coercive if 

∥z∥ → ∞ , z ∈ S =⇒ ψ(z) → ∞ .

A coercive continuous objective on a closed feasible set attains its minimum. More generally, it is enough that one lower level set is nonempty and bounded. 

# 6.3 Existence via the KKT formulation 

For the KKT reformulation, if the feasible set in ( x, y, λ )-space is closed and one lower level set is nonempty and bounded, then an optimal solution exists. For the original MPEC, a clean sufficient condition is the following. 

Theorem 6.1 (Basic existence result for MPEC) . Assume that: 

11 1. f is continuous, 2. Z is closed, 3. the lower-level data are continuous, 4. each gi(x, ·) is convex and differentiable, 5. a multiplier constraint qualification and SBCQ hold so that Gr( S) is closed, 6. some lower level set {(x, y ) ∈ F : f (x, y ) ≤ α} is nonempty and bounded. Then the MPEC has a global optimal solution. 

# 6.4 The affine/quadratic case 

In the MPAEC setting with polyhedral Z and quadratic objective f , each active-set piece is polyhedral. An analog of the Frank–Wolfe theorem says that a quadratic objective over a finite union of polyhedra attains a global solution when it is bounded below on the feasible set, equivalently when it is bounded below on every feasible ray of every polyhedral piece. 

# 6.5 Interpretation 

The hard part of existence theory for MPEC is usually not continuity of the objective. The hard part is proving that the feasible region is closed despite being generated by equilibrium constraints. Once closedness and boundedness are available, existence follows from standard compactness arguments.

Part (c): Computational Hurdles (8 points) 

The primary hurdle is that the operators z+ and z− are non-differentiable at zi = 0. Consequently, the normal map equation is non-smooth, meaning standard gradient-based or Newton methods cannot be directly applied. 

Overcoming this: One can use Semismooth Newton methods (which rely on gen-eralized Jacobians/subdifferentials), or apply a smoothing function (e.g., replacing max(0 , z i) with a smooth approximation like the Fischer-Burmeister function or a soft-plus function) to restore differentiability. 

Rubric: 4 pts for identifying non-differentiability at z = 0 . 4 pts for a valid algorithmic solution (semismooth Newton, smoothing functions, etc.).

17 Problem 4: Monotonicity and the Implicit Form 

Part (a): Strong Monotonicity (8 points) 

F (x, ·) is strongly monotone on C(x) if there exists a constant c > 0 such that for all 

u, v ∈ C(x): (u − v)T (F (x, u ) − F (x, v )) ≥ c∥u − v∥2

Since F is affine with respect to y, F (x, u ) − F (x, v ) = M (u − v). The condition becomes ( u − v)T M (u − v) ≥ c∥u − v∥2. This holds if and only if M is a positive definite matrix .

Rubric: 4 pts for the mathematical definition of strong monotonicity. 4 pts for stating 

M must be positive definite. 

Part (b): Topological Property of S(x) (8 points) 

If strong monotonicity holds, the solution set S(x) consists of exactly one element 

(it is a singleton). The reaction map is single-valued. 

Rubric: 8 pts for stating the solution is unique/a singleton. 

Part (c): The Implicit Form (9 points) 

Because S(x) is single-valued, let y(x) denote this unique solution vector. The implicit form of the MPEC is: minimize f (x, y (x)) subject to x ∈ X

Advantage: It reduces the MPEC to a standard nonlinear program (NLP) in the variable x alone, stripping away the combinatorial complexity of the complementarity constraints. 

Limitation: If there are joint constraints ( x, y ) ∈ Z (rather than just x ∈ X), the implicit function y(x) might violate these constraints. It is extremely difficult to char-acterize the feasible region of x such that ( x, y (x)) ∈ Z , which can lead to an empty feasible set or highly intractable constraints. 

Rubric: 3 pts for the correct formulation. 3 pts for the advantage (reduces to NLP). 3 pts for the limitation (handling joint constraints).
