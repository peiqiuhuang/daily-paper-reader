## 摘要
大语言模型（LLMs）通常通过更新参数（例如通过强化学习 RL）来针对下游任务进行训练。然而，更新参数会迫使模型吸收特定任务的信息，这可能导致灾难性遗忘和塑性丧失。相比之下，使用固定 LLM 参数的上下文学习（in-context learning）可以廉价且快速地适应特定任务需求（例如通过提示词优化），但其本身通常无法达到通过更新 LLM 参数所能获得的性能提升。没有充分的理由将学习限制在上下文或权重更新之中。此外，人类也可能在不同的时间尺度上进行学习（例如系统 1 与系统 2）。为此，我们为 LLMs 引入了一种快慢学习框架，将模型参数视为“慢”权重，将优化的上下文视为“快”权重。这些快“权重”可以从文本反馈中学习以吸收特定任务的信息，同时允许慢权重保持更接近基础模型并保留通用推理行为。在推理任务中，快慢训练（FST）的样本效率比仅进行慢学习（RL）高出多达 3 倍，同时始终能达到更高的性能上限。此外，经 FST 训练的模型与基础 LLM 保持更近的距离（KL 散度减少高达 70%），从而比 RL 训练产生的灾难性遗忘更少。这种减少的漂移还保留了塑性：在完成一个任务的训练后，FST 训练的模型比仅参数训练的模型能更有效地适应后续任务。在任务领域动态变化的持续学习场景中，FST 能够持续获取每个新任务，而仅参数 RL 则会陷入停滞。

## Abstract
Large language models (LLMs) are trained for downstream tasks by updating their parameters (e.g., via RL). However, updating parameters forces them to absorb task-specific information, which can result in catastrophic forgetting and loss of plasticity. In contrast, in-context learning with fixed LLM parameters can cheaply and rapidly adapt to task-specific requirements (e.g., prompt optimization), but cannot by itself typically match the performance gains available through updating LLM parameters. There is no good reason for restricting learning to being in-context or in-weights. Moreover, humans also likely learn at different time scales (e.g., System 1 vs 2). To this end, we introduce a fast-slow learning framework for LLMs, with model parameters as "slow" weights and optimized context as "fast" weights. These fast "weights" can learn from textual feedback to absorb the task-specific information, while allowing slow weights to stay closer to the base model and persist general reasoning behaviors. Fast-Slow Training (FST) is up to 3x more sample-efficient than only slow learning (RL) across reasoning tasks, while consistently reaching a higher performance asymptote. Moreover, FST-trained models remain closer to the base LLM (up to 70% less KL divergence), resulting in less catastrophic forgetting than RL-training. This reduced drift also preserves plasticity: after training on one task, FST trained models adapt more effectively to a subsequent task than parameter-only trained models. In continual learning scenarios, where task domains change on the fly, FST continues to acquire each new task while parameter-only RL stalls.

### 论文总结：快慢学习（Learning, Fast and Slow）

#### 1. 核心问题与整体含义
论文探讨了大型语言模型（LLM）在适配下游任务时的根本矛盾：
*   **参数更新（慢学习）**：如强化学习（RL）或监督微调（SFT），虽然性能强，但会导致模型吸收过多任务特定信息，引发“灾难性遗忘”和“塑性丧失”（即模型变得僵化，难以学习新任务）。
*   **上下文学习（快学习）**：如提示词优化，虽然灵活且不改变模型，但性能通常无法匹配参数更新。
*   **研究动机**：借鉴人类的双系统学习机制，提出**快慢训练（Fast-Slow Training, FST）**框架。其核心目标是让模型参数（慢权重）保持通用推理能力，而让优化的上下文（快权重）吸收任务特定的启发式信息，从而实现高效、持续且不失塑性的任务适配。

#### 2. 方法论：核心思想与技术细节
FST 框架将 LLM 适配视为两个互补组件的联合优化：
*   **慢权重（Slow Weights, $\theta$）**：即模型参数。使用带有验证奖励的强化学习（RLVR）进行更新，具体采用了 **CISPO**（一种改进的策略梯度算法）和 **GRPO** 的优势计算方法。
*   **快权重（Fast Weights, $\Phi$）**：即文本提示词池。使用 **GEPA**（反射式进化提示优化）进行更新。GEPA 会维护一个提示词种群，利用一个冻结的“反射模型”对训练过程中的错误进行自然语言批评，并生成改进的提示词。
*   **核心流程**：
    1.  **交替循环**：训练分为多个周期。在每个周期开始时，先运行快循环（GEPA），根据当前模型的表现优化出一组处于帕累托前沿（Pareto-frontier）的提示词种群。
    2.  **联合训练**：在接下来的慢循环中，固定提示词种群，通过 RL 更新模型参数。在计算优势函数时，会将不同提示词诱导的行为差异纳入考虑。
    3.  **分工协作**：快权重通过文本反馈快速捕捉任务信号，慢权重则在进化后的上下文引导下进行更稳健的参数漂移。

#### 3. 实验设计
*   **数据集/场景**：
    *   **CodeIO**：代码输出预测。
    *   **Polaris (Math)**：数学推理。
    *   **HoVer-hard**：多跳事实验证。
    *   **Star-graph**：合成的星型图搜索任务（用于测试零奖励环境下的探索能力）。
*   **Benchmark 与对比方法**：
    *   **Baseline**：标准强化学习（仅更新参数）、纯 GEPA（仅优化提示词）、FST-distill（将提示词知识蒸馏到参数中）。
    *   **评估维度**：样本效率、性能上限（渐近线）、KL 散度（模型漂移）、塑性保留（任务 A 训练后学习任务 B 的能力）、持续学习（任务 A->B->C 的连续适应）。

#### 4. 资源与算力
*   **硬件**：主要实验在单节点 **8× H100 (80GB)** 上运行；部分消融实验（如 Polaris 的大规模测试）使用了 4 个节点（32× H100）。
*   **时长与效率**：
    *   单个 RL 步骤约 60 秒，FST 步骤约 100 秒（若不使用重用技术）。
    *   通过“卷展重用”（Rollout Reuse）技术，FST 的单步耗时可降至 47 秒，比纯 RL 更快。
    *   单次完整训练任务大约消耗 **25-40 H100-GPU 小时**。
*   **模型**：主要使用 Qwen3-8B 系列模型。

#### 5. 实验数量与充分性
*   **实验规模**：论文在四个不同的任务领域进行了深入测试，涵盖了数学、代码、常识推理和合成任务。
*   **消融实验**：非常充分。作者对种群大小 $K$、优势函数基准选择、循环长度 $T$、以及提示词重用机制进行了详细的消融研究。
*   **客观性**：通过拟合 Sigmoid 曲线来比较“性能上限”而非单一时间点，并引入了 KL 散度与奖励的帕累托曲线分析，实验设计严谨且多维度，能够客观反映 FST 的优势。

#### 6. 主要结论与发现
*   **高效率**：FST 达到相同奖励水平的速度比纯 RL 快 **3 倍**。
*   **高性能**：FST 最终达到的性能上限（Asymptote）始终高于纯 RL。
*   **低漂移**：在相同性能下，FST 模型的 KL 散度比 RL 低 **70%**，意味着模型更接近原始基座，保留了更多通用能力。
*   **强塑性与持续学习**：FST 训练后的模型在面对新任务时表现出极强的适应能力，而纯 RL 训练后的模型在连续学习新任务时往往会陷入停滞（性能跌至近 0%）。

#### 7. 优点
*   **架构创新**：成功将神经科学中的“快慢权重”概念引入 LLM 后训练阶段，解决了参数更新过于“沉重”的问题。
*   **数据效率**：利用文本反馈（而非仅标量奖励）来优化提示词，极大地克服了强化学习中“每回合仅 1 比特信息”的限制。
*   **实用性强**：提出的“卷展重用”机制显著降低了计算开销，使其在实际生产环境中具有可行性。

#### 8. 不足与局限
*   **任务限制**：目前主要集中在具有**可验证奖励（Verifiable Rewards）**的推理任务上，对于主观性较强的生成任务（如创意写作）尚未验证。
*   **依赖反射模型**：FST 的效果部分取决于用于生成提示词批评的“反射模型”（实验中使用了 GPT-5.2 等强模型）的质量。
*   **系统复杂性**：相比单一的 RL 流程，FST 需要维护提示词种群并进行周期性的进化计算，系统实现复杂度较高。
