## Response-Aware User Memory Selection for LLM Personalization 

Jillian Fisher 1 Jennifer Neville 2 Chan Young Park 2

Abstract 

A common approach to personalization in large language models (LLMs) is to incorporate a sub-set of the user memory into the prompt at infer-ence time to guide the model’s generation. Ex-isting methods select these subsets primarily us-ing similarity between user memory items and input queries, ignoring how features actually af-fect the model’s response distribution. We pro-pose Response-Utility optimization for Memory 

Selection (RUMS), a novel method that selects user memory items by measuring the mutual information between a subset of memory and the model’s outputs, identifying items that re-duce response uncertainty and sharpen predic-tions beyond semantic similarity. We demonstrate that this information-theoretic foundation enables more principled user memory selection that aligns more closely with human selection compared to state-of-the-art methods, and models 400 × larger. Additionally, we show that memory items selected using RUMS result in better response quality com-pared to existing approaches, while having up to 95% reduction in computational cost. 

1. Introduction 

As large language models (LLMs) become ubiquitous in daily interactions, personalization has emerged as critical for user satisfaction. To meet this demand, current ap-proaches to personalization generally fall into two cate-gories: training-based methods and inference-time methods. Training-based approaches, such as low-rank preference models (Bose et al., 2025) or parameter-efficient fine-tuning modules (Tan et al., 2024), adapt the model parameters for individual users. Although these methods can be effective for small populations, they become computationally pro-hibitive when scaled to many users. As a result, inference-

What information, if any, should be included to guide the model? A na ¨ıve strategy is to include all stored user mem-ories in the prompt. However, this method often reduces performance, as irrelevant or distracting information may overwhelm the model and produce responses that are less accurate or sound unnatural (Li et al., 2023; Rajeev et al., 2025). Moreover, this strategy becomes increasingly imprac-tical as the memory size grows. Therefore, existing methods instead attempt to select and retrieve relevant subsets of the user memory by measuring the relationship between a mem-ory item and the input query . In other words, these methods try to pick out only the user memory items that seem most related to the user’s current request. This is often done by heuristic retrieval (L `u, 2024), surface-level semantic simi-larity (Lei et al., 2023), or prompting LLMs (Handa et al., 2024). While intuitive, we hypothesize that measuring how memory items affect output (i.e. response) distri-butions directly, rather than similarity to input queries, provides a more reliable signal for personalization . Se-lecting user memory items based on semantic similarity to the input may biases selection toward redundant, surface-aligned items and away from non-obvious but important ones, leading to both over-selection and under-selection relative to true utility. Such shortcomings can introduce noisy items that degrades response quality while increasing latency, cost, and privacy risks (Hagstr¨ om et al., 2025). To test this hypothesis, we introduce Response-Utility for Memory Selection (RUMS), which comprises of two key components: a novel utility function ( RUMS-Utility ) that quantifies the informativeness of user memory subsets, and a lightweight model ( RUMS-Model ) that efficiently approxi-mates this utility computation at inference time. 

RUMS-Utility utilizes an information-theoretic framework to directly measure how much each subset of user memo-ries reduces uncertainty in the model’s response distribution. Specifically, it computes the conditional mutual informa-tion between candidate memory subsets and the model’s output distribution given the input query. While this guarantee assumes the model’s distribution aligns with human prefer-ences, our empirical results confirm that modern LLMs are sufficiently calibrated for this entropy signal to serve as a high-fidelity proxy for true utility. However, computing RUMS-Utility for all queries is com-putationally expensive, particularly with large user profiles. Therefore, we use the utility function during an offline train-ing phase to generate high-quality training data. We then train RUMS-Models , a lightweight encoder-classifiers based on DeBERTa (He et al., 2021), to directly predict memory items that are most informative given a query. This learned approach preserves the principled selection quality of the utility function while enabling efficient deployment. 

Through extensive empirical testing, the experimental results validate our hypothesis: RUMS enables more prin-cipled memory item selection that is more closely aligned to human selection compared to state-of-the-art (SOTA) mem-ory selection methods and models 400 × its size. We also show that the items selected using RUMS result in better response quality than other methods, while lowering the cost of personalization overall. Our work offers novel conceptual, theoretical, and empirical contributions to personalization: • We propose RUMS, a lightweight, response-based method for user memory selection that directly accounts for the impact of memory items on model outputs. • We establish a principled utility function, RUMS-Utility, and provide a theoretical proof linking entropy reduction to expected utility maximization. We further validate the necessary assumption of model-human alignment through empirical results, showing that RUMS-Utility can effec-tively guide the training of a lightweight RUMS-Model. • We show that RUMS (i) reliably identifies when per-sonalization is beneficial, aligning closely with human judgments, (ii) selects memory subsets that align more strongly with human annotations than SOTA retrieval and prompting baselines, and (iii) improves generation quality over SOTA baselines and 400 × larger models on synthetic and real-world datasets, while adapting to un-derlying model capabilities and costing up to 95% less than other methods. 

2. Related Works 

User Memory for LLM Personalization Memory augmen-tation enables personalization in LLMs by retrieving subsets of user memory at inference time to guide responses. Simple approaches prepend user context to prompts, such as profile summaries (Richardson et al., 2023; Pan et al., 2025) or past interactions (Kang et al., 2023; Liu et al., 2023). However, as the amount of user memory grows, these methods quickly become intractable. Due to this, there has been an increased trend in selecting a subset of the user memory to prepend. For example, Chen (2025) retrieves relevant historical items via semantic similarity for recommendations, while LaMP (Salemi et al., 2024b) investigates personalized sentence retrieval. PEARL (Mysore et al., 2024) and ROPG (Salemi et al., 2024a) train retrieval components with user signals to improve personalization. For a broader overview, see Liu et al. (2025). However, these methods rely on the similarity between user input and memory item, without considering their impact on the model’s downstream response. 

Selective Personalization A related line of reserach is se-lective conditioning or abstention which studies how models can ignore user memory or abstain from predictions when inputs are uninformative. Geifman & El-Yaniv (2017) intro-duce selective classification for neural networks, and recent work extends this to language models, proposing mecha-2Response-Aware User Memory Selection 

nisms to ignore input features or context that may degrade performance (Hagstr ¨om et al., 2025). In memory selection, this corresponds to sometimes choosing no subset when per-sonalization is unnecessary. Prior work typically relies on global thresholds (Li, 2023) or static policies (Joren et al., 2023), whereas our method learns a sequential, adaptive policy that dynamically selects which items to condition on based on the input and context. 

Information-theoretic Uses in NLP Information-theoretic ideas have long informed NLP (Brown et al., 1992), span-ning multimodal learning (Almud ´evar et al., 2025), informa-tion control (Ravfogel et al., 2020), interpretability (Chen et al., 2018), and prompt design (Zhang et al., 2022). In feature selection, EDDI (Ma et al., 2019) estimated con-ditional mutual information for individual features, while later work extended this to dynamic or sequential selection (Covert et al., 2023; Gadgil et al., 2024). Recent methods also use differences in next-token likelihood with and with-out context to guide generation through distribution editing (He et al., 2025). While much of this work has focused on domains such as image classification and fraud detection, the closest parallel to personalization arises in recommen-dation systems using Bayesian active learning to select a model query to ask the user, with the goal of improving recommendation quality (Handa et al., 2024). We draw on these principles to select memory items that maximize information gain for personalized responses.

3.1. RUMS-Utility 

In order to formulate a surrogate utility function, we frame memory selection for personalization as an information ac-quisition problem: given a large space of user memory, the goal is to select the items that most improve our response to an input. The intuition is that a memory subset S is valuable to the extent it reduces uncertainty in the model’s predictions. For instance, for “where should I eat lunch?”, without user in-formation, the model spreads probability across countless restaurants, yielding high entropy. Therefore, the utility of the memory subset S is measured directly by reduction in the entropy. 

3.2. Theoretical Foundation 

We formalize conditions under which reducing predictive uncertainty leads to higher expected user utility. While the theoretical foundation rely on a latent-variable abstraction, we validate empirically (Section 4) that minimizing predic-tive entropy aligns with improved user-perceived utility. 

Assumption 1 (Existence of an Optimal Latent Variable) .

For each user u and query x, there exists a latent random variable Zu ∼ p(Zu | x) representing the user-specific latent target that determines what constitutes an optimal response. Interpretation and Limitations Theorem 3.1 provides a principled justification for selecting memory subsets that re-duce predictive entropy. However, in practice, this surrogate objective is only meaningful if the model’s predictive dis-tribution pθ (y|x, m u,S ) is aligned with human judgments ,that is, responses the model assigns higher probability to under relevant memory subsets tend to be more useful to hu-mans. While Theorem 3.1 provides a principled foundation, 

empirical evaluation (Section 4) is necessary to verify that reductions in entropy translate into improvements in actual user-perceived utility.

3.4. RUMS-Models: Learning to Approximate Utility 

While RUMS provides a principled approach to memory selection, explicitly computing entropy reductions over all possible subsets at inference time is prohibitive, particularly for users with large memory profiles. To enable efficient deployment, we use RUMS offline to generate training data and train a lightweight model to directly predict memory selections at inference. 4Response-Aware User Memory Selection 

Training Data Generation Our training pipeline uses RUMS to generate (user query, optimal memory subset) pairs. For each query x, we pair it with a diverse set of users 

{u1, . . . , u K }, each with memory items mu. When profiles are large ( |mu| ≥ 50 ), we first use an LLM to filter to k can-didate items. We then compute bU(S, u ) from Equation (4) for candidate subsets S, selecting the highest-utility subset 

S∗(x, u ) = arg max S bU(S, u ) if it exceeds threshold τ , or the empty set otherwise. This yields K training examples 

{(x, u k, S ∗(x, u k)) }Kk=1 per query. Pairing each query with diverse profiles encourages the model to learn genuinely informative item selection rather than spurious correlations. The threshold τ , tuned on validation data, enables automatic abstention when personalization would not help. 

Model Architecture and Training We train RUMS-Models as lightweight classifiers to approximate utility computation at inference. For dynamic memory schemas, where item types vary across users, a binary classifier (RUMS-Binary)

outputs all selections in a single pass. The resulting RUMS-Models match the inference speed of simple similarity-based retrieval while retaining principled selection quality, making it practical for large-scale deployment. 

4. Results 

In this section, we evaluate RUMS by defining three outputs: 

RUMS-Utility , the utility score from Equation (3); RUMS-Subset , the selected memory subset from Equation (4); and 

RUMS-Models , models trained on RUMS-generated data. We evaluate our approach along three dimensions: H1: RUMS-Utility provides a reliable signal of human preference for distinguishing between inputs that do and do not benefit from personalization. H2: The subsets selected, RUMS-Subset, choose memory subsets more aligned with human preference than SOTA baselines and larger models. Furthermore, the utility signal is strong enough that models trained on them (RUMS-Models) also outperform baselines and larger models. H3: Memory subsets selected with RUMS-Models im-prove downstream response generation compared to state-of-the-art baselines and larger models.

4.2. H1: Does RUMS-Utility Distinguish Personalized from Non-Personalized Inputs? 

We hypothesize that entropy reduction when conditioning on user memory reflects its importance. Thus, inputs that do not benefit from personalization should yield low maximum utility scores (i.e., the maximum utility score across all sub-sets of candidate memory items for a specific user), while personalized inputs should yield high ones. To test this, we compare the maximum utility scores on a known non-personalized (FreebaseQA) and a known personalized (Per-sonaFeedback) datasets. We sampled 500 inputs per dataset and computed the maximum utility score using Equation (3) for each input. The resulting distributions were compared using divergence metrics and hypothesis testing. 

Utility scores separate personalized from non-personalized inputs. In Figure 2a we show that personalized inputs exhibit much higher utility scores (mean = 0.50 , std = 0.12 ) than non-personalized ones (mean = 0.12 , std = 0.09 ). The Jensen–Shannon divergence between distributions is high at 0.66 , and a two-sample Kol-mogorov–Smirnov test yields a statistic of 0.82 (p < . 001 ), strongly rejecting the null hypothesis of equal distributions. These results indicate that maximum utility scores reliably separate personalized from non-personalized inputs. If both humans and the model selected no items, the sample was counted correct for both metrics. 

RUMS-Utility correlate more strongly with human judg-ments. As shown in Table 2b, RUMS-Utility outperforms GPT-4 and semantic similarity baselines. Compared to GPT-4 FewShot, it achieves +7% recall and +15% specificity overall. Gains are strongest on synthetic data, with competi-tive results on real-world. Other baselines show imbalanced behavior (e.g., GPT-4 ZeroShot often avoids personalization, giving high specificity but low recall). In contrast, utility scores perform well on both metrics, reflecting their ability to capture each item’s true effect on generation. 

4.3. H2: Does RUMS-Subset Match Human Judgment? 

Building on H1, we now ask whether our method iden-tifies which specific memory items help personalization. We randomly selected n = 15 samples from each of Per-sonaFeedback, FreebaseQA, WildChat-NonPersonal, and WildChat-Personal for a total of n = 60 . As before, four annotators selected any (or none) of 50 possible items they judged useful, yielding 41% majority ( k ≥ 3) agreement. While moderate, this level of agreement is expected given the subjective nature of the task. We therefore considered an item as selected by humans if a majority agreed, resulting in an average of 4.19 items selected per user query for inputs where at least one item was chosen. We evaluated (i) direct selection from utility scores RUMS-Subset (up to 5) and (ii) RUMS-Models, the trained models described in Section 4. We compare RUMS-Subset and RUMS-Models against GPT-4 (prompted to select up to five relevant memory items, both zero- and few-shot) and seman-tic similarity (selecting up to five highest-scoring items with cosine similarity > 0). Performance was measured with precision, recall, and F1 against human labels. 

RUMS-Subset aligns more closely with human annota-tion. As shown in Table 1, the two methods based on RUMS achieve on average 20% higher precision, 31% higher re-call, and 9% higher F1 compared to the baselines across both synthetic and real-world datasets. Semantic similar-ity performs worst, reflecting reliance on surface overlap rather than downstream utility. GPT-4 shows strong recall but substantially lower precision, indicating that it tends to over-select. In contrast, the results show that the raw utility scores, RUMS-Subset, themselves provide strong sig-nal for identifying useful memory items, as RUMS-Models achieves similar F1, demonstrating that small models with modest amounts of data can learn effective memory selec-tion from utility scores. For analysis of the approximation gap between RUMS-Utility and RUMS-Models, see Sec-tion A.3; qualitative examples appear in Section A.6.

5. Conclusion 

We introduced RUMS, an information-theoretic method for user memory selection that leverages response-aware utility as a more reliable signal for personalization than input-similarity. Our results show that RUMS accurately identi-fies when personalization is needed, selects memory subsets that align better with human judgment than SOTA base-lines, and improves downstream generation quality while reducing computational cost by up to 95% . A. Additional Experimentation 

In this section, we present additional experiments to supplement our main results. 

A.1. Robustness Analysis of Utility Scores 

In H1, we hypothesized that RUMS-Utility, the maximum utility scores, can reliably distinguish between user inputs that would benefit from personalization and those that would not. Here, we further investigate how hyperparameters in the RUMS-Utility algorithm affect this reliability and provide justification for these effects. Specifically, we examine the influence of model type, model size, model family, decoding method type, maximum number of tokens generated, number of Monte Carlo samples, and number of user profiles. Equating the two expressions and rearranging yields 

H(y | x, m u,S ) = H(y | x, z u, m u,S ) + H(zu | x, m u,S ) − H(zu | x, y, m u,S ).

By Assumption 2, memory mu,S provides little additional information about y beyond (x, z u). We therefore approximate 

H(y | x, z u, m u,S ) ≈ H(y | x, z u).

Substituting this approximation gives 

H(y | x, m u,S ) ≈ H(y | x, z u) + H(zu | x, m u,S ) − H(zu | x, y, m u,S ).

which is also independent of S.Combining these observations, we obtain the approximation

F. Experimental Details 

This section provides the full details of the experiments described in Section 4. 

F.1. Experimental Setup 

Across all experiments (H1–H3), we consider a fixed set of n = 50 memory items (e.g., location, age, dietary restrictions ). The task is to select a subset of up to n = 5 memory items that can improve personalization. 

F.2. Datasets 

To evaluate scenarios where personalization is beneficial as well as those where it is not, we use a diverse mixture of user inputs drawn from prior work, including both synthetic and real-world sources. The three datasets we used are described below: 29 Response-Aware User Memory Selection 

PersonaFeedback (Tao et al., 2025) This is a synthetically created dataset consisting of user-specific queries designed to reflect inputs from realistic and diverse user personas. First, they constructed personas by representing them as a structured dictionary of memory items, including demographics, personality traits, and preferences. Then, open-source content from social media, reviews, and forums were used to populate these items with specific values. Lastly, an LLM generator was used to produce queries based on a sampled persona and specific sub-scenarios. Additional steps such as similarity-based filtering, rephrasing to avoid directly revealing persona information, and manual annotations ensured diversity and realism. The final dataset contains more than 4,000 persona-specific questions. The original data is in Chinese and therefore, we translated to English using a machine translation model (Tiedemann & Thottingal, 2020). Example: “Do you have any music for the trip?” 

FreebaseQA (Jiang et al., 2019) A dataset of trivia-style questions and answers collected from human trivia competitions. The original dataset contains approximately 28,000 question–answer pairs. Example: “What type of snake is a sidewinder?” 

WildChat (Zhao et al., 2024) A large-scale collection of more than 1 million real-world user–ChatGPT interactions, encompassing over 2.5 million dialogue turns. The dataset was compiled by offering free access to ChatGPT and GPT-4 in exchange for user consent to share chat histories. It covers diverse conversational scenarios such as ambiguous requests, code-switching, topic shifts, and political discussions. We filter the data to retain only English conversations labeled as non-toxic, and we use only the first human-written input per conversation to ensure that queries contain sufficient context. To determine whether an input is personalized, two human annotators labeled a random selection of WildChat queries as either beneficial for personalization (WildChat-Personal) or not (WildChat-NonPersonal). Only queries with full annotator agreement were retained. Examples: “I’m hungry”; “Who is Mohamed Gamal Mohamed Maniieh?” 

Synthetic vs. Real-World Datasets By design, PersonaFeedback is treated as a personalized input set, and FreebaseQA as non-personalized. We collectively refer to these as the “Synthetic” dataset, since they were not originally created for LLM input. WildChat is considered a “Real-World” dataset, as it was collected directly from real user interactions with an LLM. We use both synthetic and real-world datasets to evaluate the robustness of RUMS across many diverse inputs. 

Sample Size by Hypothesis The number of samples varies across hypotheses. For H1, we use n = 500 samples from PersonaFeedback and FreebaseQA to analyze RUMS-Utility for binary personalization decisions. For H1’, we use n = 25 

inputs each from PersonaFeedback, FreebaseQA, WildChat-Personal, and WildChat-NonPersonal to extend the H1 analysis to correlation with human judgment. For H2, we use n = 15 inputs each from PersonaFeedback, FreebaseQA, WildChat-Personal, and WildChat-NonPersonal to examine the correlation between RUMS memory selection and human preference. Finally, for H3, we use n = 100 samples from PersonaFeedback and FreebaseQA along with n = 50 samples each from WildChat-Personal and WildChat-NonPersonal to analyze the downstream impact of subsets chosen by RUMS-Model.

Since H2 and H3 require selecting specific memory items for personalization rather than making a simple binary decision, we set an intuitive threshold of t = 0 , corresponding to no similarity between the user query and a memory item. If more than five memory items exceed this threshold, we select the top five with the strongest correlations. 

Simple Baselines: We also compare against several naive baselines. The “None” baseline provides no features ( n = 0 ), serving as a lower bound for personalized inputs and an upper bound for non-personalized ones. The “All” baseline includes all available features ( n = 50 ), testing the effect of providing maximal but noisy context. Finally, the “Random” baseline samples n = 5 items at random, establishing a chance-level reference. 

SOTA Retrieval Methods: We additionally evaluate two state-of-the-art retrieval methods. First, BM25 (L `u, 2024), a widely used ranking function in information retrieval, can be adapted for memory selection. It scores memory items (treated as terms) based on their frequency in each instance relative to the entire dataset, balancing how common a memory item is with how discriminative it is. Items with higher BM25 scores are considered more informative and are selected as the most relevant, making BM25 well-suited for sparse or high-dimensional text and categorical data. Second, ReContriever (Lei et al., 2023) is a retrieval-based method that scores memory items by their ability to help a model retrieve relevant information. It leverages dense embeddings to measure semantic similarity between queries and candidate items, effectively identifying the most informative items for downstream tasks. Memory items that yield higher retrieval relevance are prioritized, making ReContriever particularly effective for selecting contextually important or predictive inputs in text or structured datasets. Similarly to semantic similarity a threshold of t = 0 is used as threshold for selecting memory items, with a maximum of up to n = 5 selected. 33 Response-Aware User Memory Selection 

F.5. RUMS-Utility and RUMS-Model Details 

In this paper, we describe two main methods: RUMS-Utility, which selects a subset of user memory items with the highest utility for personalization directly, and RUMS-Model, a lightweight trained model built on data generated by RUMS-Utility. 

RUMS-Utility: We use RUMS-Utility not only for the analyses in H1 and H2 but also to generate training data for fine-tuning RUMS-Models. In general, we compute entropy in Equation (4) using Llama-3.1-8B (AI, 2024b) with a temperature of 0.7 and sampling-based decoding to obtain the response distribution. We restrict the maximum generation length to T = 20 tokens, chosen to mitigate bias from longer generations having naturally lower entropy. Following the algorithm in Section 3.4, we first prompt GPT-4 to reduce the candidate memory items from 50 to at least 1 and at most 10 before computing RUMS-Utility (see Prompt 7). Utility scores are then estimated using n = 5 Monte Carlo samples. For each input, we generate n = 5 distinct user profiles as separate training points, which can yield entirely unique best memory subsets of data depending on the values in the user profile. The chosen hyperparameters (e.g., model size, number of Monte Carlo samples, etc.) reflect a tradeoff: obtaining reliable signals of memory item importance while keeping computations lightweight and feasible for large-scale inference. Further discussion of these choices is provided in Section A.1.

Choosing the Personalization Threshold: To determine the threshold for deciding whether to apply personalization or select no personalization, we used a held-out validation set of size n = 100 from each of PersonaFeedback and FreebaseQA. For each query, we computed the average maximum utility score over the n = 5 generated user profiles. We labeled inputs from PersonaFeedback as personalized (1) and inputs from FreebaseQA as non-personalized (0). These scores were combined and labeled by dataset to construct an ROC curve. We then selected the threshold that maximized Youden’s J statistic ( J = TPR − FPR ), which identifies the point that best balances sensitivity and specificity. Using this procedure, we determined an optimal threshold of t = 0 .29 , which was applied in all downstream analyses. 

RUMS-Models For training the downstream models, we sampled approximately 2.5K unique PersonaFeedback queries and 2K unique FreebaseQA queries. Each query was paired with five user profiles, and RUMS-Utility was applied to select the best memory subsets, yielding 13K and 9K training examples, respectively. We excluded all FreebaseQA queries that exceeded the personalization threshold, thereby removing any noisy non-personalized samples. This design choice emphasized that non-personalized inputs should not include any memory items. We then used these (query, memory subset) pairs to train two models. The first model, used in H1–H3, was a binary logistic regression implemented with DeBERTa-v3-large (He et al., 2021) and a final linear classification layer. To train this model, we converted each (query, memory subset) pair into multiple (query, memory item, label) triplets. Training was performed with a learning rate of 5 × 10 −6 for 12 epochs, and the best model was selected based on the highest F1 score on a held-out validation set ( n = 1 K). The second model, used for H3, was a multiclass, multilabel logistic regression implemented again with DeBERTa-v2-large (He et al., 2021) and a final multiclass classification layer. Here, labels were constructed by merging the selected subsets from the n = 5 diverse user profiles per query. This produced a single label vector in which all memory items chosen in any of the five samples were marked as significant. This resulted in more dense labels, training the model to select more memory items. Training was performed with a learning rate of 2 × 10 −5 for 12 epochs, and the best model was again chosen based on the highest F1 score on a held-out validation set ( n = 1 K).

F.7. Evaluation Metric and Analysis H1 Evaluation: In H1, we compare the distribution of maximum utility scores between assumed personalized and non-personalized inputs. Specifically, we evaluate utility scores on a non-personalized dataset (FreebaseQA) and a personalized dataset (PersonaFeedback). We sampled n = 500 inputs from each dataset and computed the maximum utility score using Equation (3) for every input. The resulting distributions were then analyzed using divergence metrics and hypothesis testing. For divergence metrics, we use Jensen–Shannon (JS) divergence, a symmetric measure of similarity between two probability distributions P and Q. It is defined as: JS (P ∥Q) = 12 KL (P ∥M ) + 12 KL (Q∥M ), where M = P + Q

2

Here, KL (P ∥M ) is the Kullback–Leibler divergence from P to the average distribution M . JS divergence is always finite, symmetric, and bounded between 0 and log 2 .For hypothesis testing, we use the two-sample Kolmogorov–Smirnov test to assess whether the two samples come from the same distribution. The null hypothesis states that they do, and we test significance at the α = 0 .05 level. 

H1’ Evaluation: In H1’, performance was evaluated using human labels as the gold standard. For each sample, four human annotators rated whether n = 100 samples would benefit from personalization using a 0–2 scale. If the annotators’ confidence in personalization was 1 or higher, we considered the input to be personalized. Instructions provided to annotators are given in Prompt 8. We only included inputs with unanimous agreement ( n = 64 ) in the evaluation. The average interrater agreement between pairs of annotators was κ = 0 .59 using the Cohen’s kappa. The primary metrics used were recall, measuring accuracy in identifying personalized inputs, and specificity, measuring accuracy in identifying non-personalized inputs.

H3 Evaluation: In H3, we evaluate the downstream impact of the memory subsets chosen by each method on generated responses. For each user input, responses are generated using the query combined with the subset of memory items selected by each method. This evaluation is performed with two models: GPT-4 (OpenAI et al., 2024), a high-performing model with robust capabilities, and LLaMA 70B Instruct (AI, 2024a), a smaller model more susceptible to noise. This setup demonstrates RUMS-Models’s flexibility in adapting to models with varying capacities and usage scenarios. Performance is measured using generation win rate , defined as the proportion of pairwise comparisons in which RUMS-Models’s output is judged preferable to that of a baseline. GPT-4 is used as the judge, comparing responses generated with RUMS-Models’s selected subset against those generated with each baseline’s subset. LLM-as-Judge is validated to align closely with third-party human annotations for personalization when combined with in-context examples, model confidence scores (Dong et al., 2024). To ensure robust evaluation, our methodology includes: • In-context examples : Three examples are provided in the prompt for each evaluation to demonstrate the expected style of reasoning and scoring. • Confidence thresholds : Only responses for which the model reports a self-generated confidence score above the 70% percentile are included in the final analysis. • Chain-of-thought prompting : Evaluation prompts encourage the model to reason step-by-step and justify its preference for one response over another, improving consistency and interpretability of the win-rate comparisons. For personalized datasets (PersonaFeedback and WildChat-Personal), the evaluation metrics in the chain-of-thought include: • Completeness : Does the response fulfill the user’s request? • Personalization : Does the response appropriately incorporate relevant details from the user profile? • Overpersonalization : Does the response include irrelevant, distracting, or unnecessary user details? 36 Response-Aware User Memory Selection 

For non-personalized datasets (FreebaseQA and WildChat-NonPersonal), the metrics are: • Completeness/Accuracy : Does the response correctly and fully answer the user’s request? • Neutrality : Is the response unbiased and general, rather than tailored to the user?
