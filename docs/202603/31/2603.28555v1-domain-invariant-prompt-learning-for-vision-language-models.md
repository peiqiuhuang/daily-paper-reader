## 摘要
诸如 CLIP 之类的大型预训练视觉-语言模型通过在共享特征空间中对齐图像和文本，彻底改变了计算机视觉，并通过提示实现了鲁棒的零样本迁移。软提示（Soft-prompting）方法，如上下文优化（CoOp），通过学习一组上下文向量，有效地使这些模型适应下游识别任务。然而，CoOp 缺乏处理未见分布中领域偏移的显式机制。为了解决这个问题，我们提出了领域不变上下文优化（DiCoOp），这是 CoOp 的一种针对领域泛化进行优化的扩展。通过采用对抗训练方法，DiCoOp 强制模型学习领域不变的提示，同时保持分类的判别能力。实验结果表明，在跨越不同视觉领域的领域泛化任务中，DiCoOp 一贯优于 CoOp。

## Abstract
Large pre-trained vision-language models like CLIP have transformed computer vision by aligning images and text in a shared feature space, enabling robust zero-shot transfer via prompting. Soft-prompting, such as Context Optimization (CoOp), effectively adapts these models for downstream recognition tasks by learning a set of context vectors. However, CoOp lacks explicit mechanisms for handling domain shifts across unseen distributions. To address this, we propose Domain-invariant Context Optimization (DiCoOp), an extension of CoOp optimized for domain generalization. By employing an adversarial training approach, DiCoOp forces the model to learn domain-invariant prompts while preserving discriminative power for classification. Experimental results show that DiCoOp consistently surpasses CoOp in domain generalization tasks across diverse visual domains.

### 1. 核心问题与研究动机
*   **核心问题**：预训练视觉-语言模型（如 CLIP）虽然通过软提示（Soft-prompting，如 CoOp）在下游任务中表现出色，但这些模型在面对**领域偏移（Domain Shift）**时缺乏鲁棒性。
*   **研究动机**：现有的 CoOp 方法仅优化分类性能，未显式考虑如何提取跨领域的通用特征。在现实场景中，模型往往需要在未见的测试分布（领域泛化，DG）上保持高性能。本文旨在通过学习“领域不变”的提示词，使模型在保持分类判别力的同时，对领域变化不敏感。

### 2. 方法论
*   **核心思想**：借鉴领域对抗神经网络（DANN）的思想，在提示学习（Prompt Learning）中引入**对抗训练**。通过梯度反转层（GRL），强制上下文向量（Context Vectors）在优化分类损失的同时，最大化领域分类器的损失，从而实现领域混淆。
*   **关键技术细节**：
    *   **损失函数**：$L(v) = L_{cls}(v) - \lambda L_{dom}(v)$。其中 $L_{cls}$ 是类别分类损失，$L_{dom}$ 是领域分类损失，$\lambda$ 控制对抗强度。
    *   **梯度反转层 (GRL)**：在反向传播时将领域损失的梯度乘以 $-\lambda$，使提示词学习到无法区分领域的特征。
    *   **三种提示策略**：
        1.  **SCP (Shared Context Prompting)**：所有提示词 Token 共享，同时用于类别和领域预测。
        2.  **DFP (Domain-First Prompting)**：将提示词分为两部分，前一半为领域特定 Token，后一半为类别特定 Token。
        3.  **CFP (Class-First Prompting)**：与 DFP 相反，类别 Token 在前，领域 Token 在后。
    *   **训练流程**：每个 Batch 进行两次前向传播（类别分支和领域分支），交替更新或在特定策略下冻结部分 Token。
