## 摘要
我们研究了近期引入的视觉语言模型（VLM）领域类增量学习（domain-class incremental learning）场景。近期的研究工作利用参数高效的方法（如前缀微调 prefix-tuning 或适配器 adapters）来应对这一挑战，这些方法通过加性向量将任务特定信息融入输入标记（tokens），从而促进模型对下游任务的适配。然而，以往的方法通常对这些向量的权重进行归一化处理，忽略了不同输入标记需要不同程度调整的事实。为了解决这一问题，我们提出了动态前缀加权（Dynamic Prefix Weighting, DPW）框架，该框架动态地为前缀分配权重，并辅以适配器。DPW 由以下部分组成：1）一个门控模块，根据相应输入标记的重要性调整每个前缀的权重；2）一种加权机制，将适配器输出权重推导为前缀微调权重的残差，从而确保仅在必要时才使用适配器。实验结果表明，我们的方法在视觉语言模型的领域类增量学习场景中达到了最先进的性能。代码可在以下地址获取：https://github.com/YonseiML/dpw。

## Abstract
We investigate recently introduced domain-class incremental learning scenarios for vision-language models (VLMs). Recent works address this challenge using parameter-efficient methods, such as prefix-tuning or adapters, which facilitate model adaptation to downstream tasks by incorporating task-specific information into input tokens through additive vectors. However, previous approaches often normalize the weights of these vectors, disregarding the fact that different input tokens require different degrees of adjustment. To overcome this issue, we propose Dynamic Prefix Weighting (DPW), a framework that dynamically assigns weights to prefixes, complemented by adapters. DPW consists of 1) a gating module that adjusts the weights of each prefix based on the importance of the corresponding input token, and 2) a weighting mechanism that derives adapter output weights as a residual of prefix-tuning weights, ensuring that adapters are utilized only when necessary. Experimental results demonstrate that our method achieves state-of-the-art performance in domain-class incremental learning scenarios for VLMs. The code is available at: https://github.com/YonseiML/dpw.

*   **研究背景**：视觉语言模型（VLM，如 CLIP）在下游任务的持续学习（CL）中面临“灾难性遗忘”和“前向遗忘”（即零样本能力下降）的挑战。
*   **核心问题**：现有的参数高效微调（PEFT）方法（如前缀微调 Prefix-tuning 或适配器 Adapters）通常在**样本级别**统一分配权重，即对一个图像中的所有标记（tokens）赋予相同的任务特定信息。
*   **研究动机**：作者发现不同 token 对任务的相关性不同（例如背景 vs. 目标物体），且预训练模型的注意力投影会模糊 token 间的差异。因此，需要一种**标记级别（Token-level）**的动态加权机制，以精确控制注入的任务知识量。

### 2. 方法论：动态前缀加权（DPW）
DPW 框架通过以下三个关键技术实现精细化控制：
*   **重参数化前缀注意力（RePA）**：
    *   **核心思想**：传统方法依赖预训练的 Query-Key 投影，这会使任务相关和无关的 token 特征塌陷。
    *   **技术细节**：RePA 将复杂的 Q-K 点积简化为对输入 token 的直接仿射变换（$X W_G + B_G$），从而学习专门用于计算前缀得分的任务特定投影，保留了 token 间的区分度。
*   **条件激活（CondAct）**：
    *   **核心思想**：替代传统的 Softmax，以实现动态的总权重分配。
    *   **技术细节**：包含两个步骤：1) **条件归一化**：将总权重限制在 [0, 1] 之间，防止任务信息过度干扰预训练知识；2) **条件过滤**：基于 [CLS] token 得分的高斯分布假设，动态计算阈值并过滤掉不相关的前缀。
*   **残差加权机制（RWM）**：
    *   **核心思想**：协同前缀和适配器（LoRA）。
    *   **技术细节**：适配器的输出权重被定义为前缀权重的“残差”。只有当前缀得分超过 1 时，多出的部分才分配给适配器。这确保了适配器仅在 token 需要额外适配能力时才被激活。
