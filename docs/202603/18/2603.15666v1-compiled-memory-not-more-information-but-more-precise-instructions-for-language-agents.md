## 摘要
现有的语言智能体记忆系统主要解决记忆管理问题：即如何在上下文预算内检索和分页更多信息。我们则解决了一个互补的问题——记忆效用：哪些经验值得保留，以及它应该如何改变智能体的行为。我们提出了 Atlas，这是一个记忆内核，它能将累积的任务经验编译进智能体的指令结构中，而无需微调、RAG 或人工干预。在这里，记忆是蒸馏而非存储；交付是指令重写而非上下文注入。从智能体的失败和成功案例中提取的事实通过三步晋升门进行验证，并通过使用学习到的子要点重写智能体的系统提示词来交付。在 CUAD 合同分析任务中，进化后的提示词将 GPT-4o 的 token 级 F1 提升了 8.7 个百分点，精确率提升了 12.5 个百分点。在 HotpotQA 多跳问答任务中，联合 F1 提升了 3.16 个百分点。消融实验分离出了该机制的核心属性——训练信号约束：进化后的提示词精确地学习所教内容，不多也不少。将同样的进化提示词（由 GPT-4o 的错误编译而成，未经修改）应用于 Claude Sonnet 4.5，联合 F1 提升了 2.31 个百分点，且增益集中在 Claude 较强基准线留有最大提升空间的地方——这证实了编译后的知识是任务导向的，而非模型导向的。

## Abstract
Existing memory systems for language agents address memory management: how to retrieve and page more information within a context budget. We address a complementary problem -- memory utility: what experience is worth keeping, and how it should change agent behavior. We present Atlas, a memory kernel that compiles accumulated task experience into an agent's instruction structure -- without fine-tuning, RAG, or human intervention. Memory is distillation, not storage; delivery is instruction rewriting, not context injection. Facts extracted from agent failures and successes are verified through a three-step promotion gate and delivered by rewriting the agent's system prompt with learned sub-bullets. On CUAD contract analysis, the evolved prompt improves GPT-4o token-level F1 by $+8.7$pp and precision by $+12.5$pp. On HotpotQA multi-hop QA, joint F1 improves $+3.16$pp. An ablation isolates the mechanism's defining property -- the training signal constraint: the evolved prompt learns exactly what it is taught, and nothing more. Applied to Claude Sonnet~4.5 using the same evolved prompt -- compiled from GPT-4o errors, unchanged -- joint F1 improves $+2.31$pp, with gains concentrating where Claude's stronger baseline leaves the most room -- confirming that the compiled knowledge is task-shaped, not model-shaped.

### 论文总结：Compiled Memory (编译式记忆)

#### 1. 核心问题与整体含义
论文针对语言智能体（Language Agents）的记忆系统提出了一个根本性的挑战：**记忆的价值不在于存储容量（Management），而在于效用（Utility）**。
*   **背景**：现有的记忆系统（如 MemGPT、RAG）多关注如何检索和分页更多信息，但这往往会导致上下文噪声增加，且智能体无法自动识别哪些经验是可靠的。
*   **核心问题**：智能体能否通过累积的经验自我改进？如何将过去的失败和成功转化为能够改变未来行为的精确指令，而不是简单的上下文注入？
*   **整体含义**：提出了 **Atlas** 架构，将记忆视为一种“编译”过程——将经验蒸馏为系统提示词（System Prompt）中的结构化子要点（Sub-bullets），实现无需微调、无需人工干预的行为优化。

#### 2. 方法论
Atlas 采用了一种四层蒸馏模型和指令重写机制：
*   **四层存储结构**：
    1.  **Fresh (瞬时)**：运行范围内的原始信号。
    2.  **Task (任务)**：工作区范围的累积。
    3.  **Contextual (上下文)**：存储单次任务完成的剧集。
    4.  **Historical (历史)**：存储经过验证的事实（Fact），包含置信度、验证窗口和佐证计数。
*   **三步晋升门控（Verification Gate）**：
    1.  **去重**：通过嵌入向量相似度过滤重复项。
    2.  **LLM 验证**：批量评估候选事实的有效性。
    3.  **接地（Grounding）**：通过词汇启发式检查事实是否与源剧集共享关键关键词，防止幻觉。
*   **学习机制**：
    *   **从失败中学习**：提取“边界规则”（如：在提取法律条款时，在遇到管辖权描述前停止）。
    *   **从成功中学习**：提取“守护事实”（Guard Facts），防止在后续进化中削弱正确的行为。
*   **指令重写（Read Path）**：将高置信度（$c \ge 0.85$）的事实编译进系统提示词，替换原始提示词。这种方式在推理时**零额外成本**（不增加 RAG 检索开销）。

#### 3. 实验设计
论文在两个结构迥异的任务上进行了评估：
*   **CUAD (法律合同分析)**：
    *   **场景**：从 510 份合同中提取 41 类条款。
    *   **Benchmark**：GPT-4o 原始提示词。
    *   **对比指标**：Token 级 F1、精确率（Precision）、召回率（Recall）。
*   **HotpotQA (多跳问答)**：
    *   **场景**：跨维基百科段落进行推理。
    *   **Benchmark**：GPT-4o 原始提示词。
    *   **对比指标**：联合 F1（Joint F1，同时考核答案和支持事实的准确性）。
*   **跨模型验证**：使用 GPT-4o 错误编译出的提示词，直接应用于 **Claude 3.5 Sonnet**，验证知识的通用性。
*   **消融实验**：包括组件消融（数据量 vs 质量）和训练信号约束实验（v2 vs v3 提示词）。

#### 5. 实验数量与充分性
*   **实验规模**：

    *   使用了配对 t 检验（Paired t-tests），$p$ 值均小于 $10^{-5}$，显示出极高的统计显著性。
    *   通过跨模型（GPT-4o 到 Claude）的实验证明了该机制不是针对特定模型的“提示词工程”技巧，而是捕获了任务本身的结构，实验设计较为严谨、客观。

#### 6. 主要结论与发现
*   **性能提升**：在 CUAD 上 F1 提升 **8.7pp**，精确率提升 **12.5pp**；在 HotpotQA 上联合 F1 提升 **3.16pp**。
*   **训练信号约束（Training Signal Constraint）**：这是该机制的核心属性——系统“教什么学什么”。如果只教答案提取，支持事实的质量会下降；只有同时教两者，两者才会同步提升。
*   **任务导向而非模型导向**：由 GPT-4o 错误生成的指令能有效提升 Claude 的表现，且增益集中在 Claude 原本较弱的环节，证明编译的知识是关于“如何完成任务”的通用准则。

#### 7. 优点
*   **零推理开销**：相比 RAG，它不增加运行时的上下文长度或检索延迟。
*   **自动化闭环**：无需人工标注或手动编写提示词，实现了智能体的自我进化。
*   **高精度**：通过“边界规则”显著解决了 LLM 常见的过度包含（Over-inclusion）问题。
*   **鲁棒性**：三步验证门控有效防止了错误经验或幻觉进入长期记忆。

#### 8. 不足与局限
*   **提示词长度膨胀**：进化后的提示词长度约为原始提示词的 5 倍（约 1,570 tokens），虽然不增加检索成本，但增加了基础 Token 消耗。
*   **负样本评估缺失**：在 CUAD 实验中仅使用了正样本，可能掩盖了召回锚点（Recall Anchors）在负样本上导致误报（False Positives）的风险。
*   **接地机制较简单**：目前的关键词重叠（Keyword Overlap）检查并非真正的语义蕴含分析，可能漏掉复杂的逻辑错误。
*   **依赖分析模型**：如果用于提取事实的 LLM 本身存在严重偏见或能力不足，编译出的指令质量会受限。
